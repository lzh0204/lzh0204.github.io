JavaScript进阶知识

1.作用域

作用域（scope）规定了变量能够被访问的“范围”，离开这个“范围”变量变不能再被访问

作用域分为：全局作用域和局部作用域。其中，局部作用域又被分为函数作用域和块作用域

1.1 局部作用域

  • 函数作用域：

在函数内部声明的变量只能在函数内部被访问，在不无法直接访问

总结：

  1. 函数内部声明的变量,在函数外部无法被访问
  2. 函数的参数也是函数内部的局部变量
  3. 不同函数内部声明的变量无法互相访问
  4. 函数执行完毕后,函数内部的变量实际被清空了
  • 块作用域：

在JavaScript中使用{ }包裹的代码称为代码块，代码块内部生命的变量外部将有可能无法被访问

注意：

  1. let 声明的变量会产生块作用域,var 不会产生块作用域
  2. const 声明的常量也会产生块作用域
  3. 不同代码块之间的变量无法互相访问
  4. 推荐使用 let 或 const

1.2全局作用域

<script> 标签和 js 文件的最外层就是所谓的全局作用域，在此声明的变量在函数内部也能被访问。全局作用域中声明的变量，任何其他作用域都可以被访问

注意:

  1. 为 window 对象动态添加的属性默认也是全局的,不推荐!
  2. 函数中未使用任何关键字声明的变量为全局变量,不推荐
  3. 尽可能少的声明全局变量,防止全局变量被污染

1.3作用域链

作用域链的本质就是底层的变量查找机制

在函数被执行的时候，会优先查找当前函数作用域中的变量，如果当前作用域查找不到则会依次逐级查找父级作用域直到全局作用域

总结：

  1. 嵌套关系的作用域串联起来形成了作用域链
  2. 相同作用域链中按着从小到大的规则查找变量
  3. 子作用域能够访问父作用域,父级作用域无法访问子级作用域

1.4垃圾回收机制

垃圾回收机制（Garbage Collection） 简称 GC ，JS中内存的分配和回收都是自动完成的，内存不使用的时候会被垃圾回收器自动回收

  • 内存的生命周期：

JS环境中分配的内存，一般有以下的生命周期：

  1. 内存分配：当我们声明变量、函数、对象的时候，系统会自动为他们分配内存
  2. 内存使用：读写内存，也就是使用变量、函数等
  3. 内存回收：使用完毕，又垃圾回收器自动回收不再使用的内存

说明：

  1. 全局变量一般不会回收（关闭页面回收）
  2. 一般情况下局部变量的值，如果不用了，就会被自动回收

内存泄漏：程序中分配的内存由于某种原因程序未释放或无法释放叫内存泄漏

1.6变量提升

变量提升是JavaScript中比较“奇怪”的现象，他允许在变量声明之前即被访问（仅存在于var声明变量）

注意：

  1. 变量在未声明即被访问时会报语法错误

  2. 变量在var声明之前即被访问,变量的值为 undefined

  3. let/const 声明的变量不存在变量提升

  4. 变量提升出现在相同作用域当中

  5. 实际开发中推荐先声明再访问变量

构造函数的底层执行流程（new关键字的作用）：

当你执行 new Person('张三',20) 的时候，JavaScript 会自动完成4件事：

  1. 创建空对象：生成一个全新的空对象
  2. 绑定原型：让这个空对象的隐式原型指向构造函数的显示原型
  3. 绑定this：将构造函数的 this 只想这个空对象，执行构造函数内部逻辑（给空对象添加属性、方法）
  4. 返回实例：若构造函数没有返回对象，则自动返回第一步创建空对象

5.2实例成员和静态成员

在 JavaScript 中，实例成员和静态成员是基于是否与对象实例绑定划分的类 / 构造函数成员，核心区别在于：实例成员属于单个对象实例，静态成员属于类（构造函数本身）

  • 实例成员：

实例成员是绑定到类的具体实例对象上的属性或方法，每个实例独立拥有一份，修改一个实例的成员不会影响其他实例

  • 静态成员：

静态成员是绑定到类（构造函数本身） 上的属性或方法，不属于任何实例，所有实例共享同一份（本质是类 / 构造函数的自身属性 / 方法）

8. Promise

在讲 Promise 之前，先说一下什么是异步编程吧

异步编程是一种非阻塞的编程范式，核心是让程序在执行耗时操作（如网络请求、文件读写、数据库查询）时，不等待该操作完成，而是继续执行后续代码；当耗时操作完成后，再通过特定机制（如回调、Promise、async/await）处理其结果

简单来说就是：

  • 同步编程：代码按顺序执行，前一步完成才能执行后一步（“排队等”）
  • 异步编程：耗时操作 “后台执行”，主线程不阻塞，结果就绪后再回调（“先干别的，完事叫你”）

异步编程的核心价值是：提升程序的并发能力和响应性，充分利用 CPU 资源，避免因耗时 IO 操作浪费时间

那么异步编程的常见实现方式有：

  1. 回调函数（最基础）

将处理结果的逻辑封装成函数，作为参数传入异步操作，操作完成后调用该函数
