2.DOM概述

DOM（Document Object Model ——文档对象模型）是浏览器提供的一套专门用来操作网页内容的功能

作用：开发网页内容特效和实现用户交互

2.1DOM树

DOM树是什么：

  • 将HTML文档以树状结构直观的变现出来，我们称之为文档树或DOM树
  • 描述网页内容关系的名词
  • 作用：文档树直观的体现了标签与标签之间的关系

2.2DOM对象

概念：浏览器根据HTML标签生成的JS对象

  • 所有的标签属性都可以在这个对象上找到
  • 修改这个对象的属性会自动映射到标签身上

在HTML中是标签，而在js中是DOM对象

核心思想：把网页内容当成对象来处理

document 对象：

  • 是DOM里提供的一个对象
  • 所以它提供的属性和方法都是用来访问和操作网页内容的
  • 页面所有内容都在document里面

9.事件监听

9.1概述

就是让程序检测是否有事件产生，一旦有事件触发，就立即调用一个函数做出响应，也称为 绑定事件或者注册事件。

  • 语法：
  • 事件监听三要素：
  1. 事件源：那个DOM元素被事件触发了，要获取DOM元素（就是这里的事件对象）
  2. 事件类型：用什么方式触发，比如鼠标单击click、鼠标经过mouseover等
  3. 事件调用的函数：要做什么事

9.2鼠标事件

  • 点击相关：
  1. click：鼠标点击（左键）元素时触发
  2. dblclick：鼠标双击元素时触发
  3. contextmenu：右键点击元素时触发
  • 按键状态：
  1. mousedown：鼠标按键按下时触发（左键/右键/中键）
  2. mouseup：鼠标按键释放时触发
  • 鼠标移动：
  1. mousemove：鼠标在元素上移动时持续触发（出发频率高，需注意性能）
  2. mouseenter：鼠标进入元素时触发（不冒泡，子元素不会触发父级元素事件）
  3. mouseleave鼠标离开元素时触发（不冒泡）
  4. mouseover：鼠标进入元素或子孙元素时触发（冒泡）
  5. mouseout：鼠标离开元素或其子孙元素时触发（冒泡）

9.3焦点事件

JavaScript 中的焦点事件用于捕获元素获取或失去用户交互焦点的过程，常用于表单交互、键盘导航等场景

  1. focus:元素获取焦点时触发（不冒泡）

    触发场景：点击元素、通过tab键导航到元素、调用element.focus()等

  2. blur：元素失去焦点时触发（不冒泡）

    触发场景：点击其他元素、通过tab键离开元素、调用element.blur()等

  3. focusin：元素即将获取焦点时触发（会冒泡），相当于focus的冒泡版本

  4. focusout：元素即将失去焦点时触发（会冒泡），相当于blur的冒泡版本

13.事件流

事件流是对事件完整执行过程的描述

任意的事件被触发时总会 经历两个阶段：捕获阶段和冒泡阶段

换句话说捕获阶段就是从父到子的传导过程，冒泡阶段就是从子到父的传导过程

13.2事件冒泡

概念：当一个元素的事件被触发时，同样的事件将会在该元素的所有祖先元素中依次被触发。这一过程被称为事件冒泡

简单理解：当一个元素触发事件后，会依次向上调用所有父级元素的同名事件

事件冒泡时默认存在的，L2事件监听是第三个参数是false，或者默认都是冒泡

14.事件委托

事件委托时利用事件流的特征解决一些开发需求的知识技巧

优点：减少注册次数，可以提高程序性能

原理：事件委托其实还利用事件冒泡的特点。给父元素注册事件，当我们触发子元素的时候，会冒泡到父元素身上，从而触发父元素的事件

实现： 事件对象.target.tagName 可以获得真正触发事件的元素

注意：

  • 事件对象.target时事件实际触发的元素，他可能是直接子元素、深层后代元素，甚至就是父级元素本身
  • 在使用事件委托的时候，绑定事件的事件类型通常需要具备冒泡特性

18.节点操作

18.1 DOM节点

DOM树里每一个内容都称之为节点

常用的节点分为四类：

  • 文档节点：整个HTML文档

  • 元素节点：所有的标签，其中html是根节点

  • 属性节点：所有的属性

  • 文本节点：HTML标签中的文本内容
