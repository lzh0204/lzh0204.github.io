2. let和var的区别

在较旧的JavaScript,使用关键字 var 来声明变量,而不是let。

var 现在开发中一般不再使用它,只是我们可能再老版程序中看到它。

let为了解决 var 的一些问题。

var声明：

  • 可以先使用 再声明（不合理）
  • var声明过的变量可以重复使用（不合理）
  • 比如变量提升、全局变量、没有块级作用域

结论：var 就是个bug，别迷恋他，以后声明变量统一用let

4.类型转换

JavaScript时弱数据类型：JavaScript也不知道变量到底属于哪种数据类型，只有赋值了才清楚

使用表单、prompt获取过来的数据默认是字符串类型的，此时不能直接简单的进行加法运算。

此时需要转换变量的数据类型（把一种数据类型的变量转换成我们需要的数据类型）

4.1 隐式转换

某些运算符被执行时，系统内部自动将数据类型进行转换，这种转换称为隐式转换

规则：

  • +号两边只要有一个字符串，都会把另一边转换成字符串
  • 除了+以外的算术运算符 比如 - * / 等都会把数据类型转换成数字类型

缺点：类型转换不明确，靠经验才能总结

小技巧：

  • +号作为正号解析可以转换成数字类型
  • 任何数字和字符串相加结果都是字符串

4.2 显示转换

编写程序时过度依靠系统内部的隐式转换是不严禁的,因为隐式转换规律并不清晰,大多是靠经验总结的规律。 为了避免因隐式转换带来的问题,通常根逻辑需要对数据进行显示转换。

概念：自己写代码告诉系统该转换成什么类型

4.2.1 转换为 number 型

在JavaScript中有三种函数可以将非数值类型转换成数值类型：Number()、parseInt()、parseFloat()。

Number()可以将任意类型转换为数值类型，而后两个只能用来转化字符串类型

  1. Number()方法：

    字符串 –> 数字

    • 如果是纯数字的字符串，则直接转换为数字
    • 如果字符串内容是非数字，则转换为NaN
    • 如果字符串是一个空串或者是一个全是空格的字符串，则转换为0

    布尔 –> 数字

    • true 转换为1
    • false 转换为0

    null –> 数字

    • null转换为0

    undefined –> 数字

    • undefined转换为NaN

  2. parseInt() 方法：把一个字符串转换为一个整数

  3. parseFloat() 方法：将一个字符串转换为一个浮点数

  • &&：&&可以对符号两侧的值进行与运算并返回结果
    1. 两个值只要有一个值为false，就返回false，只有两个值都是true时才返回true
    2. JavaScript中，如果第一个值为false，则不会再检查第二个值（遇假则假）
    3. 如果两个都是true，则返回第二个值；如果两个都是false，则返回靠前的false
  • ||：||可以对符号两侧的值进行或运算并返回结果
    1. 两个值只要有一个true，就返回true，只有两个值都是false，才会返回false
    2. JavaScript中，如果第一个值是true，则不会检查第二个值（遇真则真）
    3. 如果两个都是false，则返回第二个值；如果两个都是true，则返回靠前的true
  • ！：！可以用来对一个值进行非运算，即对一个布尔值进行取反操作

5.5比较运算符

在JavaScript中有相等运算符== 、不相等运算符!=、全等运算符 ===、不全等运算符!==运算符，相同则返回true，不同返回false

  • ==：如果值的类型不同，则会自动进行类型转换，将其转换为相同类型，然后比较
  • !=：如果不相等则返回true，否则返回false，也会对变量进行自动类型转换，如果转换后相等也会返回false
  • ===：和相等运算符类似，不过他不会做自动类型转换，两个值类型不同直接返回false
  • !==：和不等运算符类似，但也不会自动类型转换，两个值类型不同，直接返回true
