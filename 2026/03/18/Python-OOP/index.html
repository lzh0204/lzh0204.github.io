Python面向对象编程

2026-03-18
Python>面向对象编程
5.3k 词

面向对象编程（Object-Oriented Programming，简称 OOP）是一种编程范式，核心思想是将程序中的数据和操作数据的方法封装成 “对象”，通过对象之间的交互来完成程序功能。

核心优势：

  1. 复用性高：继承可直接复用已有代码，避免重复编写；

  2. 易维护扩展：封装使代码模块化，修改 / 新增功能不影响整体；

  3. 结构清晰：贴合人类思维习惯，代码可读性、协作性更强。

二、面向对象的特性

在Python面向对象编程中，有三大特性：封装、继承、多态

1.封装

封装是指将对象的属性（数据）和方法（操作数据的函数）捆绑在一起，并对外部隐藏对象的内部细节，只通过公开的接口（方法）与对象交互。这样可以保护数据不被随意修改，提高代码的安全性和可维护性

在Python中通过以下方式实现封装：

  • 普通属性/方法：默认公开（可以直接访问）

  • 单下划线 _属性/方法 ：约定俗成的“私有”（提示外部不要直接访问，但可以直接访问）

  • 双下划线 __属性/方法：真正的私有（Python会做名字重整，外部不能直接访问）

  • 通过 getter/setter 方法控制属性的读写

2.继承

继承允许一个类（子类/派生类）继承另一个类（父类）的属性和方法，子类可以复写父类的代码，也可以扩展或重写父类的功能。这能减少代码冗余，体现“复用”思想

其中，Python中支持单继承和多继承

三、高级特性

1.类方法

类方法是绑定到类本身而非类的实例（对象）的方法，核心特点是不依赖具体对象的属性，可直接通过类调用，也可通过对象调用。

类方法的定义与特征：

  • 用修饰器@classmethod标识
  • 第一个参数固定为cls（代表类本身，类似实例方法的self，调用时不用手动传入）
  • 可访问 或 修改类属性（所有实例共享的属性），但不能直接访问实例属性（因为没有 self）

2.静态方法

Python 中的静态方法是属于类但不依赖于类实例 / 类本身的方法，它既不需要访问实例属性（self），也不需要访问类属性（cls），本质上就是 “挂在类名下的普通函数”

特征：

  • 静态方法通过 @staticmethod 装饰器定义

  • 没有默认参数，和普通函数的参数规则一样

  • 既可以通过类名直接调用，也可以通过实例调用

  • 无法访问类/实例的任何属性

3.特殊方法

在python中，特殊方法是以双下划线 __ 开头和结尾的内置方法（如 __init__、__str__）。它们不需要手动调用，而是在特定场景下（如创建对象、打印对象、运算符运算）由 Python 解释器自动触发，是实现面向对象特性（如封装、多态）、自定义对象行为的核心

方法 触发时机 作用
__init__ 创建对象时（类名（）） 初始化对象属性
__new__ 创建对象的第一步（比__init__早） 实例化新对象（返回新对象）
__del__ 对象被垃圾回收时 释放资源（析构）
__str__ print(对象)/str(对象) 面向用户的友好字符串
__repr__ repr(对象)/终端直接输对象 面向开发者的详细字符串
